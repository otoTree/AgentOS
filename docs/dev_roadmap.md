# AgentOS 开发路线图

根据核心功能定义，设计如下开发顺序。本顺序遵循“先基础设施，后核心能力，再扩展工具，最后渠道集成”的原则。

## 第一阶段：基础设施与核心配置 (Foundation)
这些功能是系统的基石，不完成它们，后续功能无法运行。
- **12. 模型配置**: 优先级最高，所有 AI 能力的配置中心。
- **10. RBAC 权限**: 建立多租户和权限体系，确保数据安全和隔离。
- **2. 文件系统**: 建立元索引，为 RAG、知识库和 Office 处理提供底层支撑。

## 第二阶段：核心对话与生产力工具 (Core Capabilities)
建立用户与 AI 交互的最直接入口和能力构建平台。
- **8. Super Agent**: 经典的 AI 聊天界面，作为用户的主入口。
- **3. 工作台**: 代码生成、pipeline、skill、tool 的构建平台，是系统的“技能工厂”。

## 第三阶段：数据连接与专业工具增强 (Data & Tools)
为 Agent 补充外部信息和专业处理能力。
- **6. 智能问数**: 连接数据库，为 Agent 提供结构化数据处理能力。
- **11. 搜索**: 集成外部搜索，解决模型时效性问题。
- **1. Office 四件套**: 实现对专业文档（Word/Excel/PDF/PPT）的深度解析和处理。
- **7. 浏览器**: 赋予 Agent 模拟人类网页操作的能力（登录、指纹注入等）。

## 第四阶段：自动化与复杂决策引擎 (Execution Engine)
让 Agent 从简单的对话走向复杂的任务执行。
- **4. 异步任务调度**: 支持触发器和自动执行，是 Task Agent 异步执行的基础。
- **5. Task Agent (Execution Engine)**: 实现 Agentic Delegation 模式，支持 Super Agent 动态委派任务给后台 Worker，实现并发执行与结果自动收集。

## 第五阶段：生态集成与多端触达 (Integration)
将构建好的能力发布到用户真实的工作场景中。
- **13. 渠道发布**: 将 Agent 部署至企微、钉钉、飞书等外部终端。
- **9. 企业邮箱**: 深度集成邮件收发，完成办公闭环。
- **14. AgenUI**: 为 Tool、Skill、Task Agent 等能力封装一层交互壳

---

## 设计逻辑说明
1. **依赖关系**: 先做“模型配置”才能聊天；先做“工作台”才能定义 Skill 和 Tool，供 Task Agent 调用。
2. **闭环路径**: 第二阶段完成后，系统已具备基本可用性（MVP）；第三阶段完成后，系统具备生产力；最后两阶段是增强竞争力。
3. **数据流向**: 先打通“文件系统”和“数据库”，后续的 Agent 才有数可问。
