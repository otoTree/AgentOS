# Plan: Fix File Upload URL Injection

## Objective
Resolve the `404` and `405` errors encountered when Python code attempts to upload files by injecting a specific, correct `file_upload_url` into the execution environment. This eliminates the ambiguity of path concatenation by the AI.

## Proposed Changes

### 1. `src/lib/code-wrapper.ts`
- **Update `ExecutionContext` interface**: Add optional `fileUploadUrl?: string`.
- **Update `wrapCode` function**:
    - Extract `fileUploadUrl` from context.
    - Inject it into the Python environment: `os.environ['FILE_UPLOAD_URL'] = '${fileUploadUrl}'`.
    - Expose it as a global variable: `FILE_UPLOAD_URL = os.environ['FILE_UPLOAD_URL']`.
    - Add logic to automatically inject `file_upload_url` into the `main` function inputs if `FILE_UPLOAD_URL` is set (similar to how `api_url` and `api_token` are handled).

### 2. `src/app/api/run/[deploymentId]/route.ts`
- **Construct `fileUploadUrl`**:
    - Use the existing protocol and host logic.
    - Define `fileUploadUrl = ${protocol}//${host}/api/files`.
- **Pass to `wrapCode`**:
    - Include `fileUploadUrl` in the third argument (context object) when calling `wrapCode`.

### 3. `src/lib/ai.ts`
- **Update System Prompt**:
    - Remove instructions asking the AI to manually append `/files` to `api_url`.
    - Explicitly instruct the AI to use the `file_upload_url` argument for file uploads.
    - Update the `Constrains` section to list `file_upload_url` as a system-injected parameter.
    - Update the example Python code to demonstrate using `file_upload_url` directly (e.g., `upload_url = f"{file_upload_url}?public=true"`).

## Verification Plan
1.  **Review Code**: Ensure all variable names match (`file_upload_url` in Python, `fileUploadUrl` in TS).
2.  **Test Execution**: Run a deployment that uploads a file and verify that:
    - The Python script receives the correct `file_upload_url`.
    - The `main` function signature generated by AI includes `file_upload_url`.
    - The POST request is sent to `/api/files?public=true` (not `/api/files/api` or `/api`).
